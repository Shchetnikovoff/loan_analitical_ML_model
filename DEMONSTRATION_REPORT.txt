4. СТРУКТУРА ДАТАСЕТА
================================================================================

Train данные:
  • Количество записей: 58,645
  • Количество колонок: 13 (11 признаков + id + target)

Test данные:
  • Количество записей: 39,098
  • Количество колонок: 12 (11 признаков + id)

Числовые признаки (7):
  1. person_age                 - Возраст человека
  2. person_income              - Годовой доход
  3. person_emp_length          - Стаж работы в годах
  4. loan_amnt                  - Сумма кредита
  5. loan_int_rate              - Процентная ставка
  6. loan_percent_income        - Кредит % от дохода
  7. cb_person_cred_hist_length - Длина кредитной истории

Категориальные признаки (4):
  1. person_home_ownership      - Статус владения жильём
  2. loan_intent                - Цель кредита
  3. loan_grade                 - Класс кредита (A-F)
  4. cb_person_default_on_file  - История дефолтов (Y/N)

Целевая переменная:
  • loan_status: 0 = без дефолта, 1 = дефолт

7. ПРОЦЕСС ОБУЧЕНИЯ
================================================================================

Этапы:
  1. Загрузка данных из Kaggle датасета
  2. Разделение на train/validation (80/20, stratified)
  3. Предобработка признаков (StandardScaler + OneHotEncoder)
  4. GridSearchCV с 27 комбинациями параметров, 3-fold CV
  5. Обучение лучшей модели
  6. Оценка на валидации
  7. Генерация предсказаний для теста
  8. Сохранение модели и submission

Время обучения: ~15-20 минут (GridSearchCV: 81 fit операция)
